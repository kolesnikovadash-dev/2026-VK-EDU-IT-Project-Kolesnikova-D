## Hiring Plan

| Роль | Зоны ответственности |
|---|---|
| Product Manager | Формирует roadmap, приоритизирует backlog |
| UX/UI Designer | Проектирует пользовательский путь, отрисовывает интерфейс |
| Frontend Developer | Пишет веб-приложение на React и веб-админку |
| Backend Developer | Пишет серверную логику на Django, интегрирует внешние API |
| QA Manual | Тестирует пользовательские сценарии, ищет баги |

---

## Development Framework — Scrum

- Продукт на старте требует частой проверки гипотез — короткие спринты по 2 недели позволяют быстро получать фидбек от первых байеров и корректировать направление
- Требования к расчёту стоимости и интеграциям с внешними API могут меняться — Scrum позволяет адаптироваться без остановки разработки
- Небольшая команда из 5 человек — идеальный размер для Scrum без лишней бюрократии

## Team Rituals

| Встреча | Цель | Частота |
|---|---|---|
| Daily Standup | Синхронизация команды, выявление блокеров | Ежедневно, 15 минут |
| Sprint Planning | Планирование задач на спринт, оценка в story points | Раз в 2 недели |
| Sprint Review | Демо готового функционала, сбор обратной связи | Раз в 2 недели |
| Retrospective | Обсуждение что работает и что улучшить в процессах | Раз в 2 недели |
