## Анализ рисков

| Риск | Тип | Уровень | Стратегия реагирования |
|---|---|---|---|
| Недостаточная база производителей на старте | Внутренний | Высокий | До запуска вручную собрать минимальную базу из 50-100 производителей, договориться с несколькими напрямую |
| Неточность расчёта стоимости | Внутренний | Высокий | Регулярно обновлять логистические тарифы через Freightos API, добавить дисклеймер что расчёт ориентировочный |
| Низкий спрос со стороны байеров | Внешний | Средний | Запустить с фокусом на early adopters — брендах которые уже обожглись на Alibaba, собирать фидбек и итерировать |
| Отказ Freightos API или изменение условий | Внешний | Средний | Иметь резервный источник тарифов, кэшировать последние данные чтобы платформа работала при временном отказе |
| Конкуренты копируют идею | Внешний | Низкий | Строить unfair advantage через базу данных и скоринг производителей — это не копируется быстро |
